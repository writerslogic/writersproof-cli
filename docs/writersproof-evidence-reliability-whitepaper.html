WritersProof Evidence and Reliability

How cryptographic witnessing produces verifiable proof of human authorship
White Paper · Version 1.0 · June 2026 · WritersLogic Inc.
Patent Pending: USPTO Application No. 19/460,364

1. The Problem with Detection

AI-generated text detectors operate by statistical inference: they estimate the probability that a given text was produced by a language model based on token distributions, perplexity scores, and stylometric features. These approaches share a fundamental limitation: they analyze the output without any knowledge of the process that produced it.

Detection accuracy degrades with every advance in language model capability. As models produce more human-like text, the statistical distance between human and machine output narrows, and detector false-positive rates climb. A student accused of cheating based on a detector score has no reliable way to prove the accusation wrong, because the detector's confidence is not evidence of anything except its own model's opinion.

The core asymmetry: Detection asks "does this text look like it was written by a human?" WritersProof asks "was this text observably written by a human, and can we prove it?"

2. Proof of Process: A Different Approach

WritersProof inverts the detection paradigm. Instead of analyzing finished text for statistical artifacts, it captures behavioral evidence during document creation and packages that evidence into cryptographically signed, independently verifiable packets.

The system implements the Proof-of-Process (PoP) protocol, specified in draft-condrey-rats-pop, an IETF Remote Attestation Procedures (RATS) architecture document. PoP treats authorship as an attestable claim backed by observable evidence, not a statistical inference from the text itself.

What is captured

  • Keystroke timing — inter-key intervals at nanosecond resolution, revealing the rhythm, cadence, hesitation patterns, and acceleration curves unique to each writer
  • Editing behavior — insertions, deletions, cursor movements, undo/redo sequences, and revision patterns that distinguish composition from transcription
  • Focus events — application switching, window focus changes, and document engagement duration, establishing continuous attention to the writing process
  • Mouse dynamics — movement patterns, micro-corrections, and scroll behavior that correlate with reading, reviewing, and editing phases
  • Temporal anchors — cryptographic checkpoints with verifiable delay function (VDF) proofs that bind evidence to wall-clock time, preventing backdating

What is never captured

  • The content of keystrokes (which keys were pressed)
  • The text of the document
  • Screenshots, clipboard contents, or screen recordings
  • Browsing history or activity in other applications

3. The Evidence Architecture

WritersProof evidence is structured in three layers, each independently verifiable and progressively harder to forge.

Layer 1: Behavioral Signals

Raw behavioral data is captured by the WritersProof sentinel, a background process that monitors keystroke timing, mouse dynamics, and application focus events. Signals are stored in a write-ahead log (WAL) with HMAC-SHA256 integrity tags, ensuring that entries cannot be modified after capture without detection.

Each event is signed using an Ed25519 key that is behaviorally gated: the signing key remains available only while the user is actively typing. After 30 seconds of inactivity, the key material is zeroized from memory and must be re-derived from accumulated keystroke entropy. This creates a cryptographic binding between the signing key and the physical act of writing.

Layer 2: Forensic Analysis

The forensic engine performs multi-dimensional analysis of captured behavioral signals to produce quantified assessments:

Analysis Module What It Measures Forgery Resistance
Velocity profiling Typing speed distribution, acceleration curves, fatigue signatures Requires real-time keystroke generation matching target profile
Cadence analysis Rhythm regularity, burst/pause patterns, digraph timing Statistical fingerprint unique to each writer
Cognitive load detection Hesitation clustering, revision depth, editing complexity Reflects genuine thought processes, not reproducible by replay
Transcription detection Unnaturally uniform timing, absence of revision, copy-paste ratios Distinguishes original composition from dictation or paste
Cross-modal consistency Correlation between keystroke timing, mouse behavior, and focus patterns Validates that all behavioral channels tell the same story
Jitter entropy analysis Hardware-level timing noise from CPU scheduling and interrupt jitter Cannot be reproduced in software; proves physical hardware involvement

Layer 3: Cryptographic Packaging

Evidence is packaged into PoP Evidence Packets (.cpop files), serialized as CBOR (RFC 8949) with COSE_Sign1 signatures (RFC 9052). Each packet contains:

  • A Merkle Mountain Range (MMR) root committing to all checkpoint hashes
  • VDF proof binding the evidence to elapsed wall-clock time
  • The forensic assessment scores and the raw signal summaries that produced them
  • The signing public key and checkpoint chain for independent verification
  • An EAT (Entity Attestation Token) profile identifier per RFC 9711

The wire format uses IANA-registered CBOR tag 1129336656 (0x43504F50) and media type application/vnd.writersproof.cpop+cbor.

4. Verification: Trust Without Authority

WritersProof evidence is designed for independent verification. Any party with the evidence packet can confirm its integrity without contacting WritersLogic or any central authority. Verification checks:

  1. Signature validity — COSE_Sign1 Ed25519 signature over the CBOR payload
  2. Checkpoint chain integrity — MMR proof connecting all checkpoints to the root hash
  3. Temporal consistency — VDF proof confirming minimum elapsed time between checkpoints
  4. Behavioral plausibility — forensic scores within human-normal ranges
  5. Cross-modal agreement — keystroke, mouse, and focus signals corroborate each other

5. Attestation Tiers

Not all evidence carries the same weight. WritersProof classifies evidence into three tiers based on the depth and corroborability of the captured signals:

Tier Label Requirements Confidence
1 Verified Full behavioral capture, VDF temporal proofs, multiple checkpoints, cross-modal consistency, hardware jitter entropy Highest. Evidence is independently verifiable with no reliance on trust in the author or platform.
2 Corroborated Partial behavioral capture (e.g., keystroke timing without mouse), fewer checkpoints, temporal anchors present but reduced signal depth Moderate. Consistent with human authorship but lacks full cross-modal corroboration.
3 Declared Author self-attestation with cryptographic signature but limited or no behavioral evidence (e.g., retroactive attestation of existing documents) Lowest. Proves the author claims authorship and signed the claim, but does not independently demonstrate the writing process.

6. Forgery Cost Analysis

The security of WritersProof evidence rests not on any single mechanism but on the combined cost of simultaneously defeating multiple independent verification channels. An attacker attempting to forge evidence must:

  1. Generate plausible keystroke timing at nanosecond resolution that matches a target writer's velocity profile, cadence fingerprint, and cognitive load signature. This requires modeling the target's typing behavior across thousands of data points.
  2. Produce consistent mouse dynamics that correlate with the forged keystroke timing in ways that reflect genuine reading and editing behavior.
  3. Simulate application focus patterns that match the temporal structure of the forged typing sessions, including realistic pause-and-return behavior.
  4. Generate valid VDF proofs for each checkpoint, which requires sequential computation that cannot be parallelized, binding the evidence to real elapsed time.
  5. Produce hardware-level jitter entropy that passes statistical tests for genuine CPU scheduling noise, which cannot be generated in software.
  6. Maintain cross-modal consistency across all of the above simultaneously, ensuring that no pair of signal channels contradicts any other.
The key insight: Each individual channel might be forgeable in isolation with sufficient effort. The requirement to forge all channels simultaneously and consistently raises the cost beyond practical feasibility. WritersProof's security is multiplicative, not additive.

7. What WritersProof Evidence Is (and Is Not)

WritersProof evidence is a factual record of observed behavioral signals during document creation, packaged with cryptographic integrity guarantees. It is:

  • Evidence, not a verdict. A WritersProof report presents observed data with quantified confidence. It does not declare that someone did or did not write something. Like a fingerprint analysis or DNA report, it provides data for a qualified decision-maker to evaluate.
  • Probabilistic, not absolute. Forensic scores reflect the statistical consistency of observed behavior with human writing patterns. A high score means the evidence strongly suggests human authorship; it does not guarantee it.
  • Tamper-evident, not tamper-proof. The cryptographic packaging ensures that any modification to the evidence after capture is detectable. It does not prevent an attacker from attempting to generate fraudulent evidence from scratch (though the forgery cost analysis above addresses the feasibility of that attack).
  • Privacy-preserving by design. WritersProof captures behavioral timing patterns, not content. The evidence proves how something was written without revealing what was written.

9. Reliability Considerations

False positives and false negatives

WritersProof does not produce binary "human" or "AI" classifications and therefore does not have a false-positive rate in the detector sense. Instead, it produces a forensic confidence score reflecting the strength of observed behavioral evidence. A low score does not mean the text was AI-generated; it means the behavioral evidence captured during writing was insufficient to make a strong attestation. This can occur when:

  • The writer uses dictation software (captured and labeled as dictation, not penalized)
  • The writer pastes content from external sources (paste events are detected and recorded)
  • The monitoring session was too short to accumulate meaningful behavioral data
  • The writer's typing style is unusually uniform (rare but possible)

Adversarial robustness

The system treats the user as a potential adversary. The behavioral key mechanism, VDF temporal proofs, hardware jitter entropy, and cross-modal consistency checks are all designed to resist forgery even by the document's author. This is essential for contexts where the author has an incentive to fabricate evidence of human authorship.

Long-term verifiability

Evidence packets are self-contained. They include all data and cryptographic material needed for verification, with no dependency on WritersLogic servers, APIs, or continued operation. A packet created today can be verified ten years from now by anyone with a CBOR parser and an Ed25519 signature verifier.
